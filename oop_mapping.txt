OOP CONCEPTS MAPPING – UB Student Exchange
===========================================

1. ENCAPSULATION
   
   • Item class: All fields private (name, price, sellerName, category)
   • Access only through public getters (getName(), getPrice(), etc.)
   • Price validation in constructor prevents invalid data

2. INHERITANCE (IS-A Relationship)
   
   • TextBook extends Item
   • Electronics extends Item  
   • Clothing extends Item
   • All three inherit fields and methods from Item

3. ABSTRACTION
   
   • Item is an abstract class – cannot be instantiated directly
   • getSummary() is abstract – each subclass provides its own implementation
   • Searchable interface – defines contract without implementation

4. POLYMORPHISM
   
   • getAll() returns ArrayList<Item> (holds any Item subclass)
   • Each subclass calls its own getSummary() at runtime
   • ListingService treats items as Searchable interface type

5. INTERFACE
   
   • Searchable.java defines: boolean matchesSearch(String keyword)
   • TextBook, Electronics, Clothing all implement Searchable
   • Each provides unique matchesSearch() implementation

6. COMPOSITION (HAS-A Relationship)
   
   • ListingService HAS-A ArrayList<Item>
   • User HAS-A ArrayList<Item> (postedItems)
   • Main HAS-A ListingService and ReportService

7. CONSTRUCTORS
   
   • Item(name, price, sellerName, category) – base class
   • TextBook uses super(name, price, sellerName, "Textbook")
   • User(name, studentId) – simple constructor

8. EXCEPTION HANDLING (Robustness)
   
   • InvalidPriceException – custom exception for price validation
   • Try-catch blocks in Main for NumberFormatException
   • InputHelper loops until valid input received

9. COLLECTIONS FRAMEWORK
   
   • ArrayList<Item> – stores listings
   • HashMap<String, ArrayList<Double>> – categories with prices
   • List<Item> – search results

10. PACKAGES
    
    app/        → Main, SeedData
    model/      → Item, TextBook, Electronics, Clothing, User
    service/    → ListingService, ReportService
    contracts/  → Searchable
    util/       → InputHelper
    exceptions/ → InvalidPriceException
